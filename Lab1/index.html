Preguntas

¿Cuál es la diferencia entre Internet y la World Wide Web?

  • Internet: Es una red global de redes de computadoras interconectadas que se comunican mediante protocolos estándar de Internet, como TCP/IP (Protocolo de Control de Transmisión/Protocolo de Internet). Es la infraestructura subyacente que permite la comunicación entre computadoras y otros dispositivos en todo el mundo. Internet proporciona la capacidad de transferir datos y recursos entre diferentes ubicaciones geográficas.
  • World Wide Web (WWW): Es un sistema de información en línea que permite el acceso a documentos vinculados mediante hipervínculos. La Web es solo una de las muchas aplicaciones que se ejecutan en Internet. Fue inventada por Tim Berners-Lee en 1989 y se desarrolló como un medio para compartir información a través de una red de computadoras. La WWW utiliza protocolos como HTTP (Protocolo de Transferencia de Hipertexto) para la transferencia de datos y HTML (Lenguaje de Marcado de Hipertexto) para la presentación de información.

¿Cuáles son las partes de un URL?

  • Esquema (Scheme): Indica el protocolo que se utilizará para acceder al recurso, como HTTP, HTTPS, FTP, etc.
  • Nombre de dominio (Domain Name): Es la dirección web del servidor donde reside el recurso. Puede ser un nombre de dominio como "example.com" o una dirección IP.
  • Puerto (Port): Es un número opcional que especifica el puerto del servidor al que se va a conectar. Por defecto, los navegadores web utilizan el puerto 80 para HTTP y el puerto 443 para HTTPS.
  • Ruta (Path): Especifica la ubicación del recurso dentro del servidor. Por lo general, se representa como una cadena de directorios y archivos separados por barras inclinadas ("/").
  • Consulta (Query): Es una cadena de parámetros de consulta que se utiliza para enviar información adicional al servidor. Se representa después del símbolo de interrogación ("?") y puede contener múltiples pares de clave-valor separados por "&".
  • Fragmento (Fragment): Indica una parte específica del recurso al que se debe desplazar el navegador después de cargar la página. Se representa después del símbolo de almohadilla ("#").

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Se utiliza para solicitar datos de un recurso específico. Es seguro e idempotente.
  • HEAD: Similar a GET, pero solicita solo los encabezados de respuesta, sin el cuerpo del mensaje.
  • POST: Se utiliza para enviar datos al servidor para su procesamiento. No es seguro ni idempotente.
  • PUT: Se utiliza para crear o actualizar un recurso en una ubicación específica. Es idempotente.
  • PATCH: Se utiliza para realizar modificaciones parciales en un recurso existente.
  • DELETE: Se utiliza para eliminar un recurso específico en el servidor. Es idempotente.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El método HTTP que generalmente se utiliza es el método POST. El motivo principal por el cual se prefiere POST para el envío de formularios es que este método está diseñado específicamente para enviar datos al servidor para su procesamiento. Al enviar un formulario con el método POST, los datos del formulario se incluyen en el cuerpo de la solicitud HTTP en lugar de en la URL, lo que proporciona cierta privacidad y seguridad, ya que los datos no se muestran en la barra de direcciones del navegador y no quedan expuestos en la caché del navegador o en los registros del servidor. Esto es especialmente importante al enviar datos sensibles como contraseñas.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

El método HTTP más comúnmente utilizado es el método GET. Este método se utiliza para solicitar datos de un recurso específico del servidor. Cuando escribes una URL en la barra de direcciones del navegador y presionas Enter, el navegador envía una solicitud GET al servidor correspondiente para recuperar la página web solicitada. El servidor responde con el contenido HTML de la página, que luego se renderiza y muestra en el navegador.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Significa que la solicitud fue exitosa. El código de estado 200 en HTTP se conoce como "OK" (Correcto). Indica que la solicitud del cliente se procesó correctamente y que el servidor está devolviendo el contenido solicitado.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Este error indica que el recurso solicitado no fue encontrado en el servidor, por lo tanto es responsabilidad del desarrollador corregir los errores 404 reportados por los usuarios en un sitio web. Cabe recalcar que esto puede ocurrir por varias razones, tales como cambios en la estructura del sitio, enlaces rotos o recursos eliminados. Mantener un sitio web funcional y sin errores es esencial para garantizar una experiencia positiva para los usuarios así como mantener la visibilidad en los motores de búsqueda.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que encontró un error 500 al intentar acceder al sitio. El error 500, también conocido como "Error interno del servidor", indica un problema en el servidor que impide que se procese la solicitud del usuario. Es crucial resolver este problema para mantener la funcionalidad del sitio, ya que normalmente se debe a problemas como errores de programación, configuraciones incorrectas, problemas de permisos, entre otros.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Cuando un atributo HTML5 está "depreciado" o "desaprobado" (deprecated), significa que se considera obsoleto y que su uso se desaconseja en favor de métodos o tecnologías más modernas y compatibles. Esto generalmente se debe a la introducción de estándares más nuevos, prácticas de codificación mejoradas o la eliminación de características anticuadas que pueden causar problemas de interoperabilidad o seguridad.

Algunos elementos de HTML 4 que están desaprobados en HTML5 incluyen:

  • El atributo align: Se usaba en HTML4 para alinear elementos como imágenes o tablas, pero en HTML5 se recomienda utilizar estilos CSS para la alineación.
  • El atributo frame y los elementos <frame>, <frameset>, <noframes>: Estos elementos solían usarse para crear diseños de marcos en páginas web, pero se han eliminado en HTML5 en favor de diseños más modernos y flexibles usando CSS.
  • El atributo name en el elemento <a> para crear anclas: En HTML5, se recomienda el uso del atributo id para este propósito en lugar del atributo name en elementos <a>.
  • Los atributos border, bgcolor, cellpadding, cellspacing en la etiqueta <table>: En HTML5, se recomienda utilizar CSS para el estilo y la presentación de las tablas en lugar de estos atributos, que se utilizaban en HTML4.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 representa una evolución significativa con respecto a HTML4 al introducir características nuevas y mejoradas. Entre estas se incluyen:

  • Semántica mejorada con elementos como <header> y <footer>
  • Soporte integrado para multimedia con <audio> y <video>
  • Capacidades gráficas y de animación con <canvas>
  • APIs para almacenamiento y comunicación, incluyendo geolocalización y WebSocket
  • Mejoras en formularios
  • Soporte para dispositivos móviles

Estas mejoras han transformado la manera en que se desarrollan y se interactúa con las aplicaciones web, permitiendo experiencias más dinámicas y potentes.

¿Qué componentes de estructura y estilo tiene una tabla?

Elementos de estructura

  • <table>: Define la estructura principal de la tabla.
  • <tr>: Define una fila dentro de la tabla.
  • <td>: Define una celda de datos dentro de una fila.
  • <th>: Define una celda de encabezado dentro de una fila, que normalmente se utiliza para las etiquetas de columna o fila.
  • <thead>, <tbody>, <tfoot>: Opcionalmente, se pueden utilizar para organizar visualmente la tabla en tres secciones: encabezado, cuerpo y pie.

Elementos de estilo

  • border: Define el ancho del borde de la tabla y sus celdas.
  • border-collapse: Especifica si los bordes de las celdas de la tabla deben colapsar en uno solo.
  • border-spacing: Establece el espacio entre las celdas de la tabla si border-collapse está configurado en separate.
  • width, height: Define el ancho y el alto de la tabla.
  • background-color, color: Establece el color de fondo y el color de texto de la tabla y sus celdas.
  • text-align, vertical-align: Alinea el texto dentro de las celdas horizontal y verticalmente, respectivamente.
  • padding, margin: Define el espacio interno y externo de las celdas de la tabla.
  • font-family, font-size: Establece la fuente y el tamaño de fuente del texto en la tabla.
  • border-color: Define el color del borde de la tabla y sus celdas.
  • border-style: Especifica el estilo del borde, como sólido, punteado, etc.

¿Cuáles son los principales controles de una forma HTML5?

  • Input: El elemento <input> es uno de los controles más versátiles en HTML5. Puede ser de varios tipos, como texto, contraseña, fecha, número, email, entre otros.
  • Textarea: El elemento <textarea> permite a los usuarios ingresar texto largo o comentarios en un área de texto multi-línea.
  • Select: El elemento <select> se utiliza para crear menús desplegables, donde los usuarios pueden seleccionar una opción de una lista predefinida.
  • Checkbox: El elemento <input type="checkbox"> permite a los usuarios seleccionar una o varias opciones de una lista de opciones.
  • Radio Button: El elemento <input type="radio"> se utiliza para crear botones de opción, donde los usuarios pueden seleccionar una sola opción de una lista de opciones.
  • Botón: El elemento <button> se utiliza para crear botones que los usuarios pueden hacer clic para enviar un formulario o realizar otras acciones.
  • Submit y Reset: Los botones <input type="submit"> y <input type="reset"> se utilizan para enviar y restablecer un formulario, respectivamente.
  • Campo de Archivo: El elemento <input type="file"> permite a los usuarios seleccionar y cargar archivos desde su dispositivo.

Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de vida de los sistemas de información se refiere al conjunto de etapas por las que pasa un sistema de información desde su concepción hasta su retiro. Estas etapas comúnmente incluyen la planificación, análisis, diseño, implementación, operación y mantenimiento del sistema. Por otro lado, el ciclo de desarrollo de sistemas de información se enfoca específicamente en el proceso de construcción del sistema, siguiendo un enfoque iterativo e incremental que incluye la definición de requisitos, el diseño, la implementación, las pruebas y la entrega del sistema funcional. Ambos ciclos están interrelacionados y forman parte del proceso de creación y gestión efectiva de sistemas de información. En resumen, el ciclo de vida de los sistemas de información abarca todas las etapas desde la concepción hasta la desactivación del sistema, mientras que el ciclo de desarrollo se centra específicamente en la construcción y mejora continua del sistema durante su desarrollo.
